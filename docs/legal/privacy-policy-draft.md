# Политика конфиденциальности ProjectSfera — черновик

Этот документ является техническим черновиком. Перед публикацией его должен проверить юрист и нужно заполнить реквизиты оператора данных.

## 1. Кто обрабатывает данные

Оператор: `[указать юрлицо/ИП/самозанятого]`.

Контакт для вопросов по данным: `[email поддержки]`.

## 2. Какие данные обрабатываются

ProjectSfera может обрабатывать:

- данные аккаунта Firebase/Telegram: идентификатор пользователя, имя, username, Telegram chat id;
- данные организации: название, invite code, роли участников;
- рабочие данные: проекты, задачи, дедлайны, исполнители, статусы, комментарии выполнения;
- файлы задач и файлы проекта;
- технические данные: время входа, last seen, ошибки API, IP/user-agent в инфраструктурных логах Vercel/Firebase.

## 3. Зачем нужны данные

Данные используются для:

- входа пользователя;
- работы организаций, проектов и задач;
- разграничения доступа по ролям;
- отправки Telegram-уведомлений;
- работы ИИ-агента по данным проектов, доступных пользователю;
- диагностики ошибок и защиты сервиса.

## 4. Где хранятся данные

Данные хранятся в Firebase/Google Cloud, Vercel и Cloudinary. Telegram используется для входа и уведомлений.

## 5. ИИ-агент

ИИ-агент получает контекст проектов, задач и извлечённого текста файлов в пределах доступа пользователя. Данные отправляются провайдеру LLM через OpenRouter для формирования ответа.

## 6. Срок хранения

Срок хранения определяется жизненным циклом организации и задач. После удаления организации данные удаляются каскадом через серверную операцию ProjectSfera.

## 7. Права пользователя

Пользователь может запросить:

- доступ к своим данным;
- исправление данных;
- удаление аккаунта или выход из организации;
- информацию о текущей обработке.

Запрос направляется на `[email поддержки]`.

## 8. Безопасность

ProjectSfera использует Firebase Auth, Firestore Security Rules, server-side API для защищённых операций и разграничение доступа по ролям.

## 9. Изменения политики

Политика может обновляться. Актуальная версия публикуется в репозитории/на сайте сервиса.
